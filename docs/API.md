## Category system

The proxy exposes 17 categories that Lidarr can use to select service and quality. Categories follow the pattern `music-[service][-quality]`, parsed at download time into `--service` and `--quality` flags for the SpotiFLAC CLI.

### SABnzbd categories (Lidarr download client)

| Category               | Quality             | Service          | SpotiFLAC `--quality` |
|-------------------------|---------------------|-------------------|--------------------------|
| music                   | Default             | default (tidal)   | LOSSLESS                 |
| music-flac-16           | CD Quality 16-bit   | default (tidal)   | LOSSLESS                 |

| music-lossless          | Best available      | default (tidal)   | HIRES_LOSSLESS           |

### SpotiFLAC service x quality matrix

| Service | LOSSLESS (16-bit FLAC) | HIRES_LOSSLESS (24-bit FLAC) |
|---------|--------------------------|---------------------------------|
| tidal   | Yes (FLAC 44.1/16)       | Yes (FLAC up to 192/24)         |
| qobuz   | Yes (FLAC 44.1/16)       | Yes (FLAC up to 192/24)         |
| amazon  | Yes (FLAC 44.1/16)       | Yes (FLAC up to 192/24)         |
| deezer  | Yes (FLAC 44.1/16)       | Limited availability            |

### Quality mapping

When Lidarr adds a download with a specific category, the proxy extracts service and quality:

```
music-qobuz-flac-24  ->  --service qobuz --quality HIRES_LOSSLESS
music-tidal-flac-16  ->  --service tidal --quality LOSSLESS
music-flac-24        ->  --service [default] --quality HIRES_LOSSLESS
music-amazon         ->  --service amazon --quality HIRES_LOSSLESS (default)
```

Pick one of three ways:
- **Quality-based**: `music-flac-16` or `music-flac-24` uses the default service at the desired quality.
- **Service-based**: `music-tidal` uses the best quality available on that service.
- **Combined**: `music-qobuz-flac-24` gives full control over both.

## Observability endpoints

| Endpoint    | Purpose                                                        |
|-------------|------------------------------------------------------------------|
| `/health`   | Liveness/readiness probe: checks DB connectivity, CLI binary, disk space. Returns 503 with the failed check names if unhealthy. |
| `/metrics`  | Prometheus text format. `spf_jobs_total{status,service}`, `spf_download_duration_seconds{service,quality}`, `spf_queue_depth{status}`. |

## Full configuration reference

All settings are environment variables prefixed `SPF_`.

| Variable                    | Default                         | Description |
|------------------------------|----------------------------------|--------------|
| `SPF_PORT`                   | `8484`                          | HTTP listen port |
| `SPF_API_KEY`                | (required)                      | API key for Lidarr auth |
| `SPF_OUTPUT_DIR`              | `/downloads`                    | FLAC output directory |
| `SPF_SPOTIFLAC_CLI_PATH`      | `/usr/local/bin/spotiflac-cli`  | SpotiFLAC binary path |
| `SPF_DEFAULT_SERVICE`         | `tidal`                          | Default download service |
| `SPF_DEFAULT_QUALITY`         | `lossless`                       | Default quality: `lossless`, `hires` |
| `SPF_FALLBACK_SERVICES`       | (none)                           | Comma-separated services to try once, in order, if the primary fails or its circuit breaker is open |
| `SPF_MAX_CONCURRENT`          | `3`                              | Max concurrent downloads |
| `SPF_JOB_TIMEOUT`             | `30m`                            | Max time per download attempt |
| `SPF_DB_PATH`                 | `/data/queue.db`                | SQLite database path |
| `SPF_HISTORY_RETENTION_COUNT` | `500`                            | History rows kept before opportunistic pruning; `0` disables pruning |
| `SPF_LOG_LEVEL`               | `info`                           | `debug`, `info`, `warn`, `error` |
